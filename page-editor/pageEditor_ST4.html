7. Challenges and Lessons Learned

Throughout the development of the splash screen and volunteer page, I encountered several challenges that provided valuable learning opportunities:

  • Video Background Implementation: Getting the video background to work properly across different browsers required testing and adjustments to ensure consistent behavior.
  • Diamond Grid Layout: Creating the diamond-shaped image grid using CSS grid and clip-path required careful planning and mathematical calculations to achieve proper alignment.
  • Animation Timing: Coordinating multiple animations on the splash screen to create a cohesive experience required fine-tuning of timing and sequence.
  • Responsive Design Challenges: Ensuring the diamond grid layout and other complex elements responded well to different screen sizes required extensive testing and media query adjustments.
  • Cross-Browser Compatibility: Addressing differences in browser rendering, especially for animations and clip-path, taught me the importance of progressive enhancement.

Key lessons learned:

  • The importance of planning complex layouts before implementation
  • The value of using modern CSS techniques like grid and flexbox for responsive designs
  • How to coordinate animations for a polished user experience
  • The need for thorough testing across different devices and browsers

8. Accessibility Implementation

Ensuring accessibility was a key consideration in developing both the splash screen and volunteer page:

  • Semantic HTML: Used proper heading hierarchy and semantic elements to improve screen reader navigation
  • Alternative Text: Provided descriptive alt text for all images to ensure content is accessible to visually impaired users
  • Keyboard Navigation: Ensured all interactive elements are accessible via keyboard navigation
  • Color Contrast: Maintained WCAG 2.1 AA-compliant color contrast ratios for text and interactive elements
  • Form Labels: Implemented proper form labeling and validation feedback for assistive technologies
  • Video Controls: Added fallback text for video content for users who cannot view videos
  • ARIA Attributes: Used appropriate ARIA roles and attributes where native HTML semantics were insufficient

The implementation follows WCAG 2.1 guidelines for perceivable, operable, understandable, and robust content.
