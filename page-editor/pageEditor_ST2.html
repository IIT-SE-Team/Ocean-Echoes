Technical Description for Implemented Pages

Home Page (home.html)

The Home Page serves as the main entry point for the website, focusing on ocean conservation awareness. The implementation approaches include:

  • Hero Section: Created a full-width video background with the "Life Below Water" headline overlay to create an immersive introduction to the site's theme.
  • Mission Statement: Implemented a centered text section with a clean, bordered design to introduce the site's purpose.
  • Issue Cards: Designed a grid layout of four cards highlighting critical ocean challenges using CSS Grid.
  • Interactive Carousel: Developed a custom image carousel using radio inputs and labels to create an interactive component that links to content pages by all team members.
  • Back to Top Button: Added a floating button that appears when scrolling to improve navigation on longer pages, styled using the global CSS variables.

Table Page (table.html)

The Table Page presents structured marine conservation data using multiple tables. The implementation approaches include:

  • Table Containers: Created table containers to display data effectively.
  • Status Indicators: Implemented color-coded status labels (excellent, good, moderate, poor, critical) using the blue color palette from global-styles.css.
  • Alternating Row Colors: Applied alternating background colors to table rows for improved readability of dense data.
  • Hover Effects: Added subtle background color transitions when hovering over rows to enhance user interaction.
  • Semantic Table Structure: Used proper HTML5 table elements (thead, tbody, tfoot).
  • Tooltip Integration: Implemented title attributes for region names that appear on hover to provide additional context without cluttering the interface.

Reflection - Challenges and Lessons Learned

  1. Video Background Implementation:

    Initially, the background video caused performance issues. I learned to optimize the video format and implement proper control attributes.

  2. Table Design:

    Creating tables with effective data display was challenging due to the amount of data. I implemented styling that improves readability while maintaining the table structure, using the scrollbar styling from global-styles.css.

  3. CSS Grid vs Flexbox:

    I had to decide when to use Grid versus Flexbox for different layout sections. This helped me better understand the strengths of each approach - Grid for two-dimensional layouts (like the issue cards) and Flexbox for one-dimensional control (like navigation).

  4. Carousel Implementation:

    Creating a JavaScript-light carousel was challenging. I learned to use CSS radio buttons as state controllers, which improved performance while maintaining functionality.

  5. Global CSS Integration:

    Establishing a consistent design system through global-styles.css required careful planning of variables and utility classes. This approach significantly improved development efficiency and visual consistency across pages.
