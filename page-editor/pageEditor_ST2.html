Technical Implementation

HTML Structure

For the website template, I created a basic structure that includes:

  • Header: Contains the website logo/identity and navigation bar
  • Main content area: Serves as a container for page-specific content
  • Footer: Includes links to each team member's editor page and validation information
  • Wrapper: Used to center content and create margins for better readability

Key HTML tags used in the template structure:

  • <header>: Semantic tag for the page header section
  • <nav>: Semantic tag for the navigation menu
  • <main>: Primary content container
  • <footer>: Semantic tag for the page footer
  • <div class="wrapper">: Container for centering content

Navigation Bar

I implemented a responsive navigation bar using HTML and CSS, following these techniques:

  • Used an unordered list <ul> with list items <li> for each navigation link
  • Added hover effects using the :hover pseudo-class to provide visual feedback
  • Highlighted the active page using a .active CSS class
  • Implemented CSS transitions for smooth hover animations
  • Made the navigation bar responsive using media queries to adapt to different screen sizes

Global CSS

I created a global CSS file (style.css) that defines the following:

  • Typography: Font families, sizes, and weights for consistency
  • Color palette: Primary, secondary, and neutral colors aligned with our sustainability theme
  • Layout: Grid structure and responsive design principles
  • Component styles: Buttons, links, cards, and other reusable elements

Key CSS properties and techniques used:

  • display: flex and display: grid: For creating flexible and responsive layouts
  • :root selector: For declaring CSS variables to maintain consistent colors and spacing
  • @media queries: For responsive design across different screen sizes
  • transition: For smooth hover effects and animations

Table Page

The Table Page presents organized, tabular data related to our sustainability theme, with interactive elements and visual styling to enhance user engagement.

Technical Implementation

Table Structure

I created a comprehensive table using HTML table elements:

  • Used <table>, <thead>, <tbody>, and <tfoot> for proper semantic structure
  • Included the website logo within the table using <img> tag with appropriate alt text
  • Used <th> elements for heading cells with scope attributes for accessibility
  • Implemented at least 8 rows and 4 columns of data related to our sustainability theme
  • Used rowspan and colspan attributes to create complex layouts where needed

Styling & Visual Elements

I enhanced the table with various styling techniques:

  • Applied zebra striping using :nth-child(odd) and :nth-child(even) selectors
  • Added border and shadow effects using border and box-shadow properties
  • Included special characters for icons within cells using HTML entity codes
  • Created thumbnail images in header cells with title attributes for tooltip functionality
  • Added hover effects to rows using :hover pseudo-class for better user interaction

Accessibility Features

I ensured the table is accessible by implementing:

  • aria-label attributes to provide context for screen readers
  • scope attributes on header cells to associate data cells with their headers
  • caption element to provide a title for the table
  • Sufficient color contrast for text and background elements

Challenges and Lessons Learned

Challenges

During the implementation of my assigned tasks, I encountered several challenges:

  1. Responsive Navigation: Creating a navigation bar that worked well across all screen sizes required multiple iterations. I initially struggled with the mobile view collapsing incorrectly.
  2. Table Complexity: Implementing merged cells while maintaining proper alignment and responsive behavior was challenging, especially when adding interactive elements.
  3. Maintaining Design Consistency: Ensuring all team members correctly implemented the global styles required clear documentation and communication.

Solutions & Lessons Learned

To overcome these challenges, I implemented the following solutions:

  1. For the responsive navigation, I researched and implemented a mobile-first approach using CSS media queries and flexbox, resulting in a more reliable navigation system.
  2. I solved the table complexity issues by breaking down the problem into smaller components and testing each part individually before integration.
  3. To maintain design consistency, I created detailed documentation with examples for other team members and held regular check-ins to address implementation questions.

Key lessons learned:

  • Planning the CSS architecture before implementation saves significant time later
  • Testing on multiple devices and browsers throughout development prevents major issues
  • Clear communication about shared components is essential for team projects
