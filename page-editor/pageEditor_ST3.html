3.2 Team Page (team.html)

The team page showcases all team members contributing to the OceanLife website project in an engaging and interactive format. This page implements all requirements for an effective team presentation.

Key Implementation Features:

  • Thumbnail Images: Each team member is represented by a clear thumbnail image with descriptive alt text
  • Hover Effects: Thumbnails respond to hover with multiple visual effects including:
    • Size increase (width and height expansion)
    • Border glow effect (color change and shadow)

7. Reflection - Challenges and Lessons Learned

Throughout the development process, I encountered several challenges that provided valuable learning opportunities:

  • Form Validation Complexity: Implementing comprehensive client-side validation while maintaining good user experience proved more complex than anticipated. I learned to balance immediate feedback with a non-intrusive approach that doesn't overwhelm users.
  • Cross-Browser Compatibility: The star rating system and custom form styling required extensive testing across browsers. This taught me the importance of progressive enhancement and fallback options for complex UI elements.
  • Performance Optimization: The content page's animations initially caused performance issues on mobile devices. I learned to implement throttling for scroll events and optimize the Intersection Observer implementation to improve performance.
  • Responsive Design Challenges: Ensuring the team member cards and side navigation responded well to different screen sizes required careful planning and testing of breakpoints.

Key lessons learned:

  • The importance of planning interactive elements with accessibility in mind from the beginning
  • How to create smooth, performance-optimized animations using modern JavaScript techniques
  • The value of user-centric design that provides clear feedback and intuitive interactions
  • The need for thorough testing across different devices and browsers

8. Accessibility Considerations

Ensuring accessibility was a priority throughout the development process:

  • Semantic HTML: Used proper heading hierarchy and semantic elements to improve screen reader navigation
  • Form Accessibility: Implemented associated labels for all form controls, error messages programmatically linked to inputs, and descriptive aria-labels
  • Keyboard Navigation: Ensured all interactive elements are keyboard accessible with visible focus states
  • Color Contrast: Maintained WCAG 2.1 AA-compliant color contrast ratios for text and interactive elements
  • ARIA Attributes: Used appropriate ARIA roles, states, and properties for complex interactive elements
  • Alternative Text: Provided descriptive alt text for all images
  • Focus Management: Implemented proper focus management for interactive elements

The implementation follows WCAG 2.1 guidelines to ensure an inclusive user experience for people with disabilities.
