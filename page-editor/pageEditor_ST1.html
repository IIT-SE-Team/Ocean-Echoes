3. Technical Description for Implemented Pages

3.1 Splash Screen (splash.html)

The splash screen serves as the landing page for Ocean Echoes website, providing an immersive introduction to the project's purpose while displaying team information. The implementation follows these key principles:

  • HTML structure with semantic elements for proper content organization
  • Video background with overlay for visual impact
  • CSS animations for text fading and positioning effects
  • Responsive design that works across different devices
  • JavaScript-controlled timing for automatic transition to main site

Key HTML Structure:

<div class="splash-container"> <div class="overlay"></div> <div class="splash"> <video autoplay muted loop playsinline> <source src="assets/video/splash-bg.mp4" type="video/mp4"> Your browser does not support the video tag. </video> <div class="go-up"> <h1 class="fade-in">Welcome To Ocean Echoes</h1> <h3 class="fade-in"> To conserve and sustainably use the oceans, seas, and marine resources for sustainable development. </h3> </div> <div class="team-members"> <h3>Team Members</h3> <ul class="name-list"> <li>Sandun Sudara</li> <li>Ushalka Kapugamage</li> <li>Githma Senivirathna</li> <li>Shenal Amerasinghe</li> </ul> </div> </div> </div>

The CSS implementation uses:

  • Keyframe animations for fade-in and positioning effects
  • Z-index management for proper layering of video and content
  • Flex layout for team member listing
  • Transform properties for centering content

Key CSS Animations:

@keyframes fadeIn { 0% { opacity: 0; } 100% { opacity: 1; } } @keyframes fadeOut { 0% { opacity: 1; } 100% { opacity: 0; visibility: hidden; } } @keyframes goUp { from { top: 60%; } to { top: 50%; } }

3.2 Volunteer Page (volunteer.html)

The volunteer page provides information about Ocean Echoes' volunteer programs, displays upcoming events, showcases reviews, and allows users to submit feedback. This page was designed with the following approach:

  • Section-based layout for organized content presentation
  • Grid and flexbox for complex layout requirements
  • Responsive design elements that adapt to different screen sizes
  • Interactive components for user engagement
  • Background gradients for visual appeal and section differentiation

The page is divided into four main sections:

  1. Introduction Section: Diamond-shaped image gallery with navigation links
  2. Event Section: Information about volunteer programs and upcoming events
  3. Review Section: Display of user reviews with rating system
  4. Feedback Section: Form for users to submit their feedback and ratings

Diamond-shaped Image Grid Implementation:

.img-content .container { display: grid; grid-template-columns: repeat(4, 12.5vw); grid-template-rows: repeat(4, 12.5vw); } .image:first-child { grid-column: 2/4; } .image:nth-child(2) { grid-column: 1/3; grid-row: 2/4; } .image:nth-child(3) { grid-column: 3/5; grid-row: 2/4; } .image:nth-child(4) { grid-column: 2/4; grid-row: 3/5; } .image { margin: 0; padding: 0; padding-top: 25vw; overflow: hidden; position: relative; clip-path: polygon(50% 0%, 100% 50%, 50% 100%, 0% 50%); }

7. Challenges and Lessons Learned

Throughout the development of the splash screen and volunteer page, I encountered several challenges that provided valuable learning opportunities:

  • Video Background Implementation: Getting the video background to work properly across different browsers required testing and adjustments to ensure consistent behavior.
  • Diamond Grid Layout: Creating the diamond-shaped image grid using CSS grid and clip-path required careful planning and mathematical calculations to achieve proper alignment.
  • Animation Timing: Coordinating multiple animations on the splash screen to create a cohesive experience required fine-tuning of timing and sequence.
  • Responsive Design Challenges: Ensuring the diamond grid layout and other complex elements responded well to different screen sizes required extensive testing and media query adjustments.
  • Cross-Browser Compatibility: Addressing differences in browser rendering, especially for animations and clip-path, taught me the importance of progressive enhancement.

Key lessons learned:

  • The importance of planning complex layouts before implementation
  • The value of using modern CSS techniques like grid and flexbox for responsive designs
  • How to coordinate animations for a polished user experience
  • The need for thorough testing across different devices and browsers

8. Accessibility Implementation

Ensuring accessibility was a key consideration in developing both the splash screen and volunteer page:

  • Semantic HTML: Used proper heading hierarchy and semantic elements to improve screen reader navigation
  • Alternative Text: Provided descriptive alt text for all images to ensure content is accessible to visually impaired users
  • Keyboard Navigation: Ensured all interactive elements are accessible via keyboard navigation
  • Color Contrast: Maintained WCAG 2.1 AA-compliant color contrast ratios for text and interactive elements
  • Form Labels: Implemented proper form labeling and validation feedback for assistive technologies
  • Video Controls: Added fallback text for video content for users who cannot view videos
  • ARIA Attributes: Used appropriate ARIA roles and attributes where native HTML semantics were insufficient

The implementation follows WCAG 2.1 guidelines for perceivable, operable, understandable, and robust content.
